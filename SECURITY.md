# Безопасность

- Движок используется в режиме Evergreen и должен регулярно получать исправления Microsoft.
- Ошибки TLS-сертификатов не обходятся и не принимаются автоматически.
- Доступ сайтов к чувствительным возможностям требует разрешения пользователя.
- Взаимодействие локальных страниц с приложением принимается только от точного
  HTTPS-origin `https://nexus.local`: стартовая страница и страница результатов
  имеют отдельные списки допустимых сообщений.
- Встраивание произвольных host objects в страницы отключено.
- Стандартные Chromium DevTools не содержат Nexus AI: отдельное расширение и
  HTTP loopback-мост для него удалены. Выбор узла для вкладки Elements использует
  только штатный локальный DevTools Protocol текущего WebView2.
- Расширения импортируются в управляемую локальную папку. Устанавливайте только проверенные расширения: они способны читать и изменять содержимое сайтов в пределах запрошенных ими разрешений.
- Встроенная блокировка трекеров не заменяет антивирус, обновления Windows и осторожность при загрузке файлов.
- AI-компоненты release-сборки находятся в папке `AI` и не загружаются во время работы. Runtime-код проверяется CI на отсутствие известных внешних адресов моделей.
- HTTP-клиенты локальных AI-runtime не используют proxy и redirect, принимают
  только loopback IP literal и устанавливают сокет непосредственно к нему.
  `llama-server` требует случайный ключ текущего процесса, переданный через
  окружение; Whisper использует новый случайный inference-маршрут при каждом
  запуске. Ключи, маршруты и порты не записываются в рапорты.
- Crawl Engine принимает только публичные HTTP/HTTPS-адреса на стандартных
  портах. В прямом режиме DNS разрешается и проверяется внутри обработчика
  соединения, после чего сокет открывается к проверенному IP; локальные и
  служебные диапазоны, metadata endpoints и DNS-rebinding блокируются. При
  пользовательском proxy безопасность удалённого DNS также зависит от proxy.
- Dependabot еженедельно создаёт обновления NuGet и GitHub Actions. Патчи автоматически объединяются только после успешной Windows-сборки; защищённая ветка должна требовать workflow `Verify Windows build`.
- Обновления `llama.cpp`, `whisper.cpp` и весов моделей не объединяются как обычные файлы: для них требуется сверка лицензии, контрольной суммы и тестов качества/безопасности модели.
- Nexus Guardian проверяет ECDSA-подпись релизного манифеста и SHA-256 критических
  файлов до запуска. Полная проверка дополнительно хэширует крупные модели;
  критическое несовпадение блокирует запуск, а некритическое включает safe mode.
- Crash Vault по умолчанию локален. Отчёт очищается от URL, истории, DOM, cookies,
  токенов, введённых значений, сетевого отпечатка, аудио и снимков экрана. В коде
  нет встроенного GitHub-токена или стороннего telemetry SDK.
- Matrix access token запрещено помещать в раздаваемую сборку. Тестовые сборки
  используют HTTPS Nexus Guardian Relay; токен бота хранится только в окружении
  сервера. Встроенный ingest key даёт лишь право отправить ограниченный отчёт,
  не предоставляет доступ к Matrix и может быть быстро заменён.
- Приватный ECDSA-ключ Guardian и Authenticode PFX никогда не хранятся в git.
  Процедура выпуска описана в `docs/Nexus-Guardian.md`.

При обнаружении уязвимости не публикуйте пароли, cookies, токены или личную историю в открытом отчёте.
