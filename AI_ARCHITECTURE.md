# Nexus Intelligence Fabric

## Граница компонентов

- `Nexus.Intelligence.Contracts` — стабильный открытый MIT-контракт.
- `Nexus.Intelligence.Fabric` — открытая реализация исследовательской логики.
- `NexusMonach` — оболочка, WebView2, UI и безопасная управляемая навигация.
- `INexusFabricHost` — узкий мост к локальным моделям и embeddings.

Все три проекта находятся в одном репозитории, собираются одной командой и
проходят один Windows CI. Модульная граница сохранена для тестов, развития и
возможных альтернативных реализаций, а не для сокрытия исходного кода.

Текущий набор операций: внутренний Smart Omnibox/Nexus Search, глубокий анализ,
углублённый поиск, итоговая сводка агента и маршрутизация локальных моделей.
Deep Page QA, Information Value Radar и дальнейшее связывание графа развиваются
в том же открытом Fabric-проекте.

Пользовательский исследователь называется Nexus Следопыт. Один Crawl Engine
обслуживает общий поиск и каталоги максимум двумя параллельными сетевыми задачами;
граф знаний сохраняет связи между запросом, источниками/товарами и выбранной
страницей без cookies, форм и полного HTML.

Глубокий анализ получает один очищенный документ. Углублённый поиск получает
до семи документов одного origin, собранных управляемой навигацией публичной
оболочки. Сводка агента получает этот же ограниченный набор и собственные
предыдущие выводы. Fabric не получает объект WebView2 и поэтому не
может самостоятельно переходить по ссылкам, нажимать элементы или читать
профиль браузера.

## Принципы

- Ни одна модель не скачивается во время работы браузера.
- Текст страницы, аудио, изображения и граф знаний не отправляются во внешние AI API.
- DOM и содержимое сайтов всегда считаются недоверенными данными.
- Модели не нажимают кнопки, не оформляют заказы и не получают пароли, cookies, токены и тела запросов.
- Тяжёлые компоненты загружаются по требованию и выгружаются после задачи.

## Распределение задач фирменным оркестратором

Пользователь не выбирает модель и не переключает технические режимы. Один запрос
поступает в Nexus Intelligence Fabric, после чего Router разбивает его на
специализированные стадии и запускает только нужные локальные компоненты.

| Компонент | Задачи | Жизненный цикл |
|---|---|---|
| Qwen3 0.6B | перевод, извлечение JSON, сравнение товаров, DevTools-пояснения, план безопасной навигации | одна текстовая задача за раз |
| Whisper base q5_1 | локальное распознавание звука активного видео | только во время перевода видео |
| multilingual-e5-small | векторы вкладок, Smart Capsules, граф знаний, смысловой поиск | низкоприоритетный фон |
| SmolVLM 500M Q8 | описание фото и визуальные признаки товара/интерфейса | только при выборе изображения |

Текстовый и речевой runtime имеют независимые очереди. Поэтому Whisper может готовить следующую
реплику, пока текстовая модель переводит предыдущую, но две тяжёлые текстовые задачи не борются
за память одновременно.

## Комплекты

`source` содержит весь собственный код и проверки. `Full Offline` дополнительно содержит папку `AI`
с проверенными native runtime и весами. Отсутствие файла показывается пользователю явно;
скрытой сетевой установки и облачного fallback нет.

AI pack устанавливается в исходное дерево только через `scripts/Install-OfflineAiPack.ps1`.
Скрипт требует `checksums.sha256`, блокирует небезопасные относительные пути и сверяет каждый файл.
GitHub workflow Full Offline берёт pack из Release этого же проекта, после чего создаёт готовый архив.

## Патчи безопасности

NuGet и GitHub Actions проверяются Dependabot еженедельно. Семантические patch-обновления могут
перейти в auto-merge только после Windows build. Native AI runtime и веса требуют ручной проверки
лицензии, SHA-256, антивирусного сканирования и набора локальных регрессионных тестов.
