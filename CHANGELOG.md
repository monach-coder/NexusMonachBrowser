# Журнал изменений Nexus Monach

Все заметные изменения проекта фиксируются здесь. Для каждой версии указываются
новые возможности, исправления, влияние на приватность, совместимость и сборку.

## 2.7.2 — в разработке

### Nexus Следопыт: непрерывное исследование и локальная диагностика — 20 июля 2026

- Контекст выбранного результата теперь живёт до ухода с сайта: переходы по
  разделам того же домена отменяют устаревшую задачу и автоматически обновляют
  выжимку в прозрачной боковой панели.
- Guardian Center получил локальный журнал Следопыта с этапом, длительностью и
  числом кандидатов/результатов. Запросы, URL, DOM, тексты страниц и личные
  данные в журнал не записываются и по сети не отправляются.
- Последовательные результаты одного запроса и домена в течение двух часов
  объединяются в одну исследовательскую сессию графа знаний вместо россыпи
  несвязанных записей.
- Извлечение фотографий товара поддерживает `srcset`, lazy-load, `picture` и
  фоновые изображения. Для видимой карточки Следопыт сначала пытается получить
  миниатюру из уже загруженного DOM, сохраняя текстовый результат при ошибке CDN.
- В crash breadcrumbs сохраняются только фиксированные технические стадии
  запуска, завершения или сбоя Следопыта — без содержимого запроса и страницы.

### GitHub Actions: Node.js 24-совместимый CI — 20 июля 2026

- Служебные `actions/checkout`, `actions/setup-dotnet` и `actions/setup-node`
  обновлены до версий с runtime Node.js 24. Предупреждение GitHub Actions об
  устаревшем Node.js 20 больше не должно появляться.
- Версия Node.js 22.17.0, закреплённая для сборки автономных AI-компонентов,
  не изменена. Автоматический кэш менеджера пакетов в `setup-node` явно
  отключён, поэтому состав и порядок Full Offline сборки остаются прежними.

### Nexus Guardian: безопасное применение обновления ядра — 20 июля 2026

- Evergreen WebView2 продолжает тихо скачиваться и устанавливаться штатной
  службой Microsoft Edge Update; Nexus не создаёт второй обновляющий процесс.
- Когда новое ядро уже готово, основное окно и Guardian Center предлагают
  перезапуск. После согласия обычные вкладки, прокрутка и допустимые непарольные
  поля сохраняются в одноразовом файле, зашифрованном Windows DPAPI для текущего
  пользователя, и восстанавливаются после запуска.
- Приватные окна, пароли, OTP, платёжные и банковские поля, данные входа и файлы
  никогда не попадают в снимок. Авторизация остаётся в защищённом профиле
  WebView2 и не копируется в отдельный JSON.
- Если зашифрованный снимок создать не удалось, Guardian отменяет перезапуск и
  оставляет окно открытым, исключая молчаливую потерю заполненных данных.

### Nexus Guardian: мониторинг WebView2 — 20 июля 2026

- Guardian Center показывает активную и установленную версии Evergreen WebView2 Runtime,
  время последней локальной проверки и версию WebView2 SDK.
- Событие `NewBrowserVersionAvailable` сообщает, когда уже установленное ядро
  начнёт использоваться после полного перезапуска браузера.
- Guardian не скачивает обновления, не меняет системную политику и не отправляет
  сведения об устройстве: обслуживание Evergreen остаётся за Microsoft Edge Update.

### UTF-8 и Google OAuth — 19 июля 2026

- Исправлена кодировка русского перевода страниц, выделенного текста и видеосубтитров.
- Для процесса Nexus OPUS явно установлена кодировка UTF-8.
- Исправлено ложное предупреждение Anti-Phishing при авторизации YouTube через
  официальные Google OAuth-адреса; защита от похожих поддельных доменов сохранена.

### Надёжность Full Offline CI — 19 июля 2026

- Full Offline workflow переведён на PowerShell 7 (`pwsh`). Unicode-строки
  сквозного теста перевода передаются через UTF-8 Base64, а наличие русского
  результата проверяется Unicode-диапазоном. Это устраняет ошибку парсера
  Windows PowerShell 5.1 до начала загрузки закреплённых AI-компонентов.
- Имена файлов в динамических URL моделей OPUS заключены в явное выражение
  PowerShell. Суффикс `?download=true` больше не принимается за часть имени
  переменной и не превращает корректный адрес Hugging Face в ответ 404.
- Guardian в CI получает абсолютный путь portable-комплекта. Служебные команды
  создания и проверки манифеста при ошибке завершаются кодом вместо показа
  недоступного графического окна; добавлены диагностические стадии и лимит
  времени, исключающий бесконечно зависшую release-сборку.

### Локальный Центр Nexus Guardian — 19 июля 2026

- В основном меню появился локальный Центр Guardian: он показывает состояние
  проверки целостности, безопасный режим и очищенные crash-report из Crash Vault.
- Рапорт можно открыть, скопировать, экспортировать в JSON или удалить. Кнопка
  диагностического теста проверяет локальный конвейер без падения браузера.
- Для новых профилей включён режим «хранить только локально». В этом режиме
  сетевые параметры скрыты, отчёты никуда не отправляются и остаются только на
  компьютере владельца.
- Формат локального отчёта оставлен совместимым с будущим серверным приёмником,
  поэтому позднее доставка подключится без миграции Crash Vault.
- Обычная локальная сборка теперь один раз создаёт отдельный ключ разработчика
  в исключённой из Git папке `.guardian-key`, подписывает манифест portable-
  комплекта и выполняет полную самопроверку до упаковки ZIP. Приватный ключ не
  попадает ни в архив, ни в репозиторий; официальный релиз использует отдельный
  защищённый ключ CI.
- В локальном Центре Guardian появилась кнопка полной проверки: она пересчитывает
  SHA-256 всех файлов, включая крупные локальные модели, и показывает результат
  без командной строки. Быстрая проверка критических файлов по-прежнему
  выполняется автоматически перед каждым запуском браузера.
- Нарушение подписи или целостности до старта браузера теперь сохраняется как
  очищенный локальный integrity-рапорт в общем Crash Vault. Одинаковая проблема
  записывается не чаще одного раза в сутки, поэтому повторные запуски не
  засоряют журнал.
- Сценарии сборки теперь явно ожидают завершения GUI-launcher Guardian через
  `Start-Process -Wait`. Это устраняет гонку Windows PowerShell 5.1, при которой
  проверка могла начаться до фактического создания локального ключа или
  подписанного манифеста.
- Проверка путей манифеста нормализует каталог запуска с завершающим `\` и без
  него одинаково. Исправлена ложная блокировка, при которой корректные файлы
  portable-сборки ошибочно объявлялись «недопустимыми путями».
- Разделены контуры доверия ключей: официальный Guardian принимает только
  встроенный публичный ключ проекта, а локальная сборка проверяет манифест
  отдельным ключом разработчика. Постоянный ключ релиза больше не конфликтует с
  локальным `RUN_BUILD.cmd`.

### Nexus Guardian Relay и Matrix — 18 июля 2026

- Добавлена доставка очищенных crash-report напрямую в обычную закрытую
  Matrix-комнату для личной установки. Access token хранится в Windows
  Credential Manager, а не в `settings.json`; E2EE-комната распознаётся и
  блокируется, чтобы не отправить в неё открытый текст.
- Для раздаваемых тестовых сборок добавлен отдельный открытый
  `Nexus.Guardian.Relay`: браузер отправляет отчёт только на HTTPS relay, а
  Matrix token остаётся на сервере владельца проекта.
- Relay ограничивает тело 128 КиБ, проверяет схему, применяет rate limit,
  сворачивает повторяющиеся сбои и не создаёт базу с IP или полными отчётами.
- Full Offline сборка умеет включить подписанный адрес relay и отдельный
  ограниченный ingest key через GitHub variable/secret. Matrix token никогда не
  попадает в архив браузера.

### Nexus Guardian — 17 июля 2026

- Добавлен отдельный `NexusMonach.exe` launcher: перед запуском внутреннего
  `NexusMonach.Browser.exe` он проверяет ECDSA P-256 подпись манифеста, SHA-256
  критических файлов и размеры крупного AI-комплекта.
- Критическое изменение блокирует запуск; изменение некритического файла и три
  аварийных завершения за десять минут включают безопасный режим без AI и
  расширений. Полная проверка хэширует также крупные модели.
- Добавлен локальный Crash Vault для managed/WPF, WebView2 и native-process
  сбоев. Отчёт хранит технические версии, этап и очищенный стек, но не историю,
  URL, запросы, DOM, cookies, токены, поля форм, IP/DNS, аудио или скриншоты.
- В настройках доступны «спросить перед отправкой», «автоматически анонимно» и
  «только локально». Без явно заданного HTTPS-приёмника ничего не отправляется.
- Full Offline workflow создаёт манифест после Authenticode, подписывает его
  отдельным секретом и выполняет полную проверку перед публикацией. Приватные
  ключи исключены из git; добавлены скрипты генерации ключа и манифеста.

### Translation Engine 3.0 — 17 июля 2026

- Захват системного звука больше не останавливается на время распознавания:
  один WASAPI-сеанс непрерывно наполняет ограниченную очередь из двух сегментов.
  Тишина отбрасывается локально, очередь вытесняет устаревший звук и не растит
  потребление памяти при медленном компьютере.
- При наличии whisper-server.exe модель Whisper загружается один раз и остаётся
  в памяти до закрытия браузера. Старый whisper-cli.exe сохранён только как
  совместимый резерв для прежних AI-комплектов.
- Из видеоперевода удалён ошибочный режим Whisper --translate: он переводил
  речь исключительно на английский. Теперь Whisper распознаёт исходный язык,
  а отдельный постоянный OPUS-конвейер выполняет перевод на русский.
- Соседние аудиоокна перекрываются на 750 мс, а повторившиеся слова удаляются
  перед переводом. Статус больше не прыгает по кругу между «слушаю» и
  «загружаю модель»; русский текст остаётся в единственном слое поверх видео.

### Специализированный автономный перевод — 16 июля 2026

- Qwen полностью исключён из перевода страницы, выделенного текста и видео.
  Его интерактивный служебный вывод больше не может попасть в DOM или субтитры.
- Full Offline комплект получил три закреплённых OPUS-MT/ONNX движка:
  многоязычный текст сначала переводится в английский, затем в русский;
  для корейского предусмотрен отдельный маршрут ko-en, а речь видео после
  Whisper использует только быстрый этап английский → русский.
- Переводчик работает одним постоянным процессом, загружает модели по требованию
  один раз, обрабатывает DOM небольшими пакетами и применяет каждый готовый пакет
  сразу. Страница, контекстное меню выделения и видеосубтитры используют один
  проверяемый сервис.
- Readiness, source-сборка и установщик AI Pack отдельно проверяют наличие OPUS-
  моделей. При неполном комплекте показывается точная ошибка вместо ложного
  прогресса или многоминутного ожидания.
- Full Offline workflow перед публикацией действительно запускает все маршруты
  перевода и требует русский результат для английской, немецкой, японской и
  корейской контрольных фраз; одних лишь существующих файлов теперь недостаточно.

### Исправления перевода и карточек — 16 июля 2026

- Карточки товаров появляются сразу после первой релевантной DOM-карточки и
  продолжают обновляться во время прокрутки и обхода каталога. Итоговое
  нейросетевое сравнение больше не задерживает первоначальный список.
- Перевод страницы использует небольшие OPUS-пакеты; успешные фрагменты сразу
  заменяются в DOM, а повреждённый фрагмент не отменяет готовую часть страницы.
- Whisper локально нормализует многоязычную речь видео в английский, после чего
  OPUS-MT en-ru выполняет перевод на русский. Исходные WebVTT и
  типовые DOM-субтитры временно скрываются, поэтому на видео остаётся один слой
  с явной меткой `RU`.
- Лимит генерации перевода рассчитывается по длине исходного текста. Это
  сокращает зависания маленькой модели и не позволяет ей продолжать ответ
  служебными пояснениями после готового перевода.

### Новые возможности

- Адресная строка снова сразу открывает выбранную обычную поисковую систему.
  После выбора результата Nexus Следопыт без скрытой навигации читает открытую
  страницу и до шести релевантных разделов того же сайта. Ход работы и выжимка
  появляются в постоянной прозрачной боковой панели; при уходе на другой домен
  панель закрывается, а исследование остаётся в локальном графе знаний.
- Добавлена отдельная кнопка SSH-терминала. Терминал использует системный Windows
  OpenSSH, явно показывает выбранный порт и длительность сессии, не хранит пароль
  и выполняет только команды, введённые пользователем.
- Карточки найденных товаров получили фотографии, взятые только из фактически
  прочитанной разметки или JSON-LD сайта. Если каталог не изменился после поиска
  либо совпадение с запросом слишком слабое, Следопыт не подменяет результат
  несвязанными товарами главной страницы.
- Qwen теперь может работать через один loopback-only `llama-server.exe` и
  остаётся загруженным между локальными задачами. Это убирает повторную заставку
  `Loading model` и заметно сокращает задержку анализа; старые
  AI-комплекты сохраняют совместимый резерв через `llama-cli.exe`.
- Перевод страницы показывает словарные совпадения сразу, а неизвестные фразы
  применяет небольшими готовыми пакетами по мере ответа локальной модели.
- Исследователь получил имя **Nexus Следопыт**. Под одним интерфейсом работает
  единый ограниченный Crawl Engine: максимум две сетевые задачи одновременно,
  до пяти внешних источников для общего поиска и до пяти страниц каталога для
  товарного режима — без запуска скрытых WebView и без второй копии моделей.
- Граф знаний теперь сохраняет исследовательские сессии: запрос, найденные
  источники или товары и выбранный пользователем результат. Источники связываются
  отдельными рёбрами «исследование»; приватные окна и запросы с признаками
  пароля, OTP, email или длинного номера в граф не записываются.
- В настройках поиска добавлены Bing и независимый Mojeek. Для всех провайдеров
  показаны ориентировочные оценки релевантности, объёма и жёсткости фильтрации с
  пояснением, что провайдер лишь обнаруживает стартовые URL.
- Перевод страницы получил мгновенный офлайн-словарь распространённых элементов
  интерфейса на английском, китайском, японском, французском и немецком. Неизвестный
  текст переводится одним крупным проходом модели, а не пословной подстановкой.
  Переведённые целые фразы повторно используются только в памяти текущего запуска
  и не создают на диске ещё одну историю просмотренного текста.
- Если выбранная поисковая система не отвечает или возвращает нечитаемую разметку,
  Crawl Engine последовательно пробует ограниченный резерв DuckDuckGo, Bing и
  Mojeek; это не меняет выбранную настройку и не запускает параллельные индексы.

- Nexus Следопыт в товарном режиме ждёт фактического изменения DOM после отправки
  поисковой формы, обходит до пяти страниц и ленивую подгрузку, распознаёт
  JSON-LD/microdata/типовые карточки и ранжирует точную фразу, полную комбинацию
  слов, словоформы и частичные совпадения до обращения к локальной модели.
- Nexus Search очищает разговорные команды вроде «открой» и «покажи» перед
  обнаружением источников; семантический и генеративный этапы имеют отдельные
  лимиты времени и всегда уступают место быстрой проверяемой выдаче, а не
  обрывают весь поиск.
- Внутренняя выдача Nexus Search сохранена как исследовательский компонент, но
  больше не задерживает обычную навигацию адресной строки и не заменяет интерфейс
  выбранной пользователем поисковой машины.
- После открытия результата исследовательский агент автоматически сопоставляет
  текст выбранной страницы с исходным запросом и показывает локальную выжимку.
  Выбор источника улучшает ранжирование похожих запросов только на устройстве.
- Агент поиска товаров получил два входа — текст и фотографию — и одну кнопку
  «Начать поиск». Итог ограничен пятью карточками с фактическими ссылками,
  ценой, рейтингом и числом покупателей/отзывов, если сайт их предоставил.
- Nexus Intelligence Fabric стал единым маршрутизатором четырёх локальных
  специализаций: Qwen (текст), Whisper (речь), multilingual-e5 (семантика) и
  SmolVLM (изображения); модели загружаются только для своей подзадачи.
- Добавлены публичный контракт и полностью открытая реализация `Nexus Intelligence Fabric™`.
- Единый запуск агента объединяет внутренние стадии глубокого анализа,
  ограниченного углублённого поиска и итоговой сводки; технические режимы не
  перекладываются на пользователя.
- Сводка агента отделяет выводы от источников, уверенности, противоречий и
  недостающей информации.
- Fabric перенесён в основное исходное дерево и собирается вместе с браузером;
  отдельный приватный репозиторий, токен release и динамическая загрузка DLL удалены.
- Добавлены необязательная Authenticode-подпись официальной сборки и проверка
  отсутствия секретов и неподтверждённых бинарных файлов в CI.

### Приватность и безопасность

- Официальные страницы входа Google в региональных доменах и Microsoft Online
  больше не вызывают ложное предупреждение Anti-Phishing; в предупреждениях
  никогда не отображаются query-параметры и идентификаторы сессии URL.
- Фоновое чтение поисковых результатов ограничено публичными HTTP(S)-адресами,
  размером ответа, временем и числом страниц; локальные, loopback-адреса,
  нестандартные порты, формы, cookies и учётные данные не передаются агенту.
- Служебный вывод `llama.cpp` теперь отсекается до передачи в DOM, субтитры,
  DevTools и внутреннюю выдачу. Повреждённый JSON локальной модели очищается от
  неэкранированных управляющих символов и имеет безопасный резервный результат.
- Перевод выделения больше не перестраивает DOM до окончания инференса, а
  меняет только выбранный текстовый узел; при ошибке оригинал остаётся на месте.
- Исследовательский агент работает только после явного действия пользователя.
- Углублённый поиск не покидает исходный origin, не открывает ссылки входа,
  профиля, корзины, оплаты и оформления заказа, не заполняет формы и не нажимает
  кнопки от имени пользователя.
- DOM и текст источников считаются недоверенными данными; инструкции со страниц
  не управляют локальной моделью.
- Fabric получает только очищенный текст, заголовок и URL ограниченного числа
  документов. Cookies, токены, пароли, значения форм и WebView2 ему не передаются.
- Fabric получает доступ к локальным моделям только через ограниченный интерфейс
  и не получает WebView2, профиль браузера или сетевые полномочия.

### Лицензирование и структура

- Название Nexus Monach, Nexus Intelligence Fabric и эмблема остаются предметом
  отдельной политики обозначений; регистрация товарного знака не заявляется.

### Сборка и совместимость

- Full Offline комплект теперь обязательно содержит `llama-server.exe`; source-
  сборка по-прежнему работает без моделей и ясно сообщает об их отсутствии.
- Восстановлена исходная фирменная заставка с пульсирующей эмблемой.
- Системный loopback-захват теперь сводит в mono не только stereo, но и 5.1/7.1
  HDMI-устройства Windows. Субтитры прикрепляются к координатам видео фиксированным
  DOM-слоем без изменения CSS контейнера плеера, что устраняет повреждение GPU-кадра.
- Перевод динамических React/Vue-страниц повторно находит актуальный текстовый
  узел, если сайт заменил DOM во время локального инференса; счётчик показывает
  только действительно применённые переводы.

- Перевод видео использует только loopback системного выхода Windows: удалён
  нестабильный захват `captureStream` внутри WebView2, вызывавший сбой
  видеорендерера. Переведённая реплика размещается у нижнего края контейнера
  видео и остаётся там в полноэкранном режиме; ошибки показываются в этом же
  контейнере без отдельного модального окна.
- Перевод страницы объединён в крупные пакеты и больше не перезапускает модель
  отдельно для каждой пропущенной строки; лимит видимого текста уменьшает время
  первого результата на компьютерах с 16 ГБ памяти.
- При переводе звука исходные дорожки субтитров временно скрываются, Whisper
  автоматически определяет многоязычную речь, а остановка loopback-захвата
  корректно снимает обработчики и освобождает аудиопоток.
- Версия приложения и Full Offline workflow обновлены до 2.7.2.
- Исправлено озвучивание названия при запуске: браузер выбирает русский мужской
  голос с безопасными резервными вариантами, явно использует устройство вывода
  Windows по умолчанию и записывает выбранный голос в `Data/startup-audio.log`.
- Обычная исходная и Full Offline сборки используют один и тот же открытый Fabric.
- Для сборки больше не требуются `NEXUS_FABRIC_TOKEN`, приватный release или
  `fabric.manifest.json`.
- Исправлены nullable-предупреждения форматирования сводки агента, которые
  блокировали Windows-сборку с параметром `-warnaserror`.
